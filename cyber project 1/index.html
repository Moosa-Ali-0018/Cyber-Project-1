Vulnerability Audit Notebook

HTML Element / Attribute Security Risk Context Defensive Remediation (Fix)
form action="http://..." Data sent over unencrypted HTTP can be intercepted via MitM (Man-in-the-Middle) packet sniffing. Always enforce https:// and utilize HSTS headers.
input type="password" Masks input on screen, but leaves data unencrypted in transit if SSL/TLS is missing. Ensure end-to-end encryption and look for DOM-based data leaks.
autocomplete="on" If left enabled on sensitive forms, browsers cached credentials locally, increasing local theft risks. Explicitly set autocomplete="off" for banking or high-security inputs.
Missing Content-Security-Policy Allows unauthorized scripts to inject directly into the HTML structure (Cross-Site Scripting). Implement strict HTTP CSP headers to restrict unauthorized script sources.

📌 Analyst Note

Phishing sites rely entirely on cloning the HTML/CSS structure of legitimate businesses. Defense includes checking domain registration age, SSL certificate transparency logs, and checking source HTML files for irregular external script targets.
